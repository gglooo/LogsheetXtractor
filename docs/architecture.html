Architecture

High-level architecture overview of LogsheetXtractor, including data flow between frontend, backend API, OCR providers, and storage.

Frontend

  • Web UI for template editing, alignment, processing, and proofreading.
  • Communicates with backend HTTP API and receives status updates.

Backend API

  • Exposes endpoint layer for templates, logsheets, ROIs, residuals, and exports.
  • Coordinates async processing pipeline and persistence.
  • Integrates with OCR/HTR script execution and external credentials.

Data and Storage

  • Relational metadata (templates, logsheets, extracted values, validation state).
  • File/object storage for uploaded files, images, and exports.

Main Workflows

1) Template Preparation

  1. Upload blank template PDF.
  2. Define ROIs and optional residual regions.
  3. Attach ROI semantic type and validation conditions.
