Frontend Guide

User-oriented walkthrough of the main UI flows in LogsheetXtractor.

Before You Start

  • Application is running and backend API is reachable.
  • OCR credentials are available (configured in the settings page).
  • You have a template PDF and sample logsheet scans.

Workflow Overview

  1. Create or open a template.
  2. Define regions of interest (ROIs).
  3. Upload logsheets, align them with the template, and start logsheet processing.
  4. Proofread extracted values and review warnings.
  5. Export the final data.

1) Templates Management

  • Browse existing templates from the templates list.
  • Create a new template or open existing one for editing.
  • Clone a template to create a new template from an existing template.
  • Delete templates that are no longer needed.
  • Preview basic information about the template (number of ROIs, number of logsheets, presence of backside).

2) Template Editor

  • Draw, move, resize, and delete ROI rectangles.
  • Autodetect ROIs from the uploaded template.
  • Name fields and assign ROI type.
  • Assign optional validation conditions to ROIs.
  • Autosave template changes.

3) Alignment (optional)

  • Every logsheet is automatically aligned upon upload.
  • Inspect and adjust alignment points manually for problematic scans.
  • Drag and drop alignment points to adjust their position until the ROIs are correctly aligned.

4) Upload and Process Logsheets

  • Upload one or more scanned logsheets to a template working space.
  • Make sure all logsheets are aligned with the template (previous step).
  • Start single or batch logsheet processing.
  • For a processed logsheet, access the proofreading interface to review and correct the extracted values.
  • For a completed logsheet, inspect or export the data.

5) Proofreading

  • Review flagged values first (validation warnings).
  • Correct OCR output where needed.
  • Mark logsheet as complete after verification.

6) Export

  • Export extracted values in an Excel spreadsheet format.
